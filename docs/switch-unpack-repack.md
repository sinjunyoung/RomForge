# Switch - 언팩 / 리팩

Switch 롬을 분해한 뒤 다시 조립(리팩)할 수 있는 툴입니다. 용량 최적화, 한글 패치 적용 등에 사용합니다.

LayeredFS 방식을 쓰면 용량이 두 배로 늘어나 스토리지 낭비가 심합니다. 이 기능을 사용하면 그런 낭비 없이 처리할 수 있습니다.

!!! warning "예외 있음"
    간혹 오히려 용량이 몇 배로 늘어나는 경우가 있어서, 모든 상황에 100% 유효한 방법은 아닙니다.

## 화면 구성

이 툴은 단독 타이틀 하나를 기준으로 동작하며, 여러 게임을 동시에 추가하는 구조가 아닙니다.

1. **파일 리스트**
2. **파일 추가** — 여러 개 동시 지정 가능
3. **폴더 추가** — 폴더 내 전체 파일 추가
4. **선택 삭제**
5. **전체 삭제**
6. **한글 패치 / xdelta 폴더 지정** — xdelta 폴더로 쓸 때는 그 안에 다른 파일이 섞여 있으면 안 됩니다.

8. **작업 폴더** — 언팩 루트 폴더이자, 최종 NSP가 생성될 폴더
9. **언팩** — 롬 파일을 분해
10. **리팩** — 분해한 폴더를 NSP로 다시 패킹

## 세부 동작

### 13. 지원 언어

체크된 언어는 사용 가능한 언어라는 뜻입니다. 체크를 해제하면 해당 언어를 강제로 비활성화합니다. 특정 언어를 비활성화하면 게임 내부에서 그 언어로 설정하는 것 자체가 불가능해질 수 있습니다.

언어를 선택하면 해당 언어의 이미지, 타이틀명, 배급사를 한 세트로 지정할 수 있습니다.

### 14. 이미지 설정 / 추출

윈도우 탐색기에서 RomForge로 이미지를 드래그하면 설정되고, 반대로 RomForge에서 윈도우 탐색기로 드래그하면 이미지가 추출됩니다.

지원 확장자: PNG, JPG, GIF, BMP, WEBP

### 17. 멀티롬 인덱스 설정

하나의 NSP 안에 여러 게임이 들어 있는 멀티롬(예: 그란디아 등)의 경우, 인덱스를 지정한 뒤 리팩하면 해당 인덱스의 게임만 별도 NSP로 생성할 수 있습니다.

### 18. 강제 언어 설정

언어를 고른 뒤 강제 언어 설정을 적용하면 그 언어를 제외한 나머지 언어는 모두 비활성화됩니다.

일본어를 한글로 패치한 게임의 경우 미국 영어가 있으면 그쪽이 우선 활성화되는 경우가 있습니다. 게임 내부에 언어 설정 기능이 없을 때 이 옵션을 쓰면 별도 설정 없이 바로 한글로 나오게 할 수 있습니다.

### 19. 강제 언어 취소

18번에서 적용한 강제 언어 설정을 되돌립니다.

## 사용 예시

**본편 + 업데이트 + DLC를 합칠 때**

여기에 추가로 한글 패치까지 합치려면, 한글 패치 폴더는 `romfs`가 보이는 상위 폴더를 지정해야 합니다.

**xdelta 지원**

폴더에 `abc.xdelta`가 있으면 `romfs` 안의 `abc.*` 전체 파일을 대상으로 패치를 시도합니다. 원본과 정확히 일치하지 않는 파일은 자동으로 무시되고, 일치하는 파일만 패치됩니다.

!!! danger "xdelta 폴더는 xdelta 파일만"
    xdelta 폴더를 지정할 때 다른 파일이 섞여 있으면 그 파일도 함께 패키징돼 버립니다. 한글 패치 폴더와 겸용으로 쓰는 구조라 자동으로 걸러내는 기능은 없으니, xdelta 폴더에는 xdelta 파일만 넣어야 합니다.

**xdelta 파일 매칭 규칙**

원본 파일명이 `battle_tex.en`이라면, xdelta 파일은 `battle_tex.en.xdelta`라는 이름이어야 매칭되어 패치됩니다. `romfs`, `exefs` 같은 세부 폴더 구조를 맞출 필요 없이 루트에 두어도 하위 전 폴더를 뒤져서 알아서 찾아 패치합니다.

같은 이름의 파일이 여러 개 있으면 원본 체크 단계에서 걸러지고, 정확히 일치하는 원본만 패치됩니다.

**용량 줄이기**

언팩 후 사용하지 않는 언어 관련 데이터를 삭제하고 리팩하면 불필요한 용량을 줄일 수 있습니다.
